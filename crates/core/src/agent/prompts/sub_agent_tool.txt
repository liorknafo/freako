- **sub_agent**: Spawn a sub-agent to handle a well-defined subtask independently. The sub-agent has access to all tools except sub_agent. Use for: exploring parts of the codebase, performing small focused refactors, researching a specific question. The sub-agent returns a summary of its findings/actions. You can call multiple sub_agents in parallel — they share the same approval state, so tools approved once are approved for all agents.
